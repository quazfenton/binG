## Step 1: Security and Access Control Hardening
- `.github/workflows/jarvis.yml`
  - Gate workflow to trusted actors (author_association check) and/or internal PRs.
  - Reduce permissions to minimum required.
  - Fix BOT_PATTERN regex to match literal `\[bot\]` suffix.
- `app/api/sandbox/webcontainer/route.ts`, `app/api/sandbox/devbox/route.ts`
  - Require authentication; remove `allowAnonymous` and `anonymous` fallback.
  - Validate startCommand with existing `validateSandboxCommand` (or block arbitrary commands entirely).
  - Ensure sessions are scoped per authenticated user and enforce ownership before command execution.
  - Guard WebContainer usage in server runtime (return clear error if invoked server-side) or move to client-only flow.
- `lib/backend/websocket-terminal.ts`, `app/api/backend/route.ts`
  - Add sandbox ownership checks (session lookup vs userId).
  - Disable host-shell terminals by default in production; only enable with explicit env flag.
  - Prefer provider PTY routes for isolated terminal access.

- `lib/api/opencode-v2-session-manager.ts`
  - Fix global quota units (minutes vs ms).

  - Remove stopped sessions from `sessions` + `userSessions` maps.

## Step 3: VFS + Context Pack Correctness

- `app/api/filesystem/context-pack/route.ts`
  - Restore absolute path validation in query schema; reject traversal or rely on VFS normalize with clear error.
  - Return generic error to client; log detailed error server-side.
- `lib/virtual-filesystem/context-pack-service.ts`
  - Enforce `maxTotalSize` with truncation warning and `hasTruncation` flag.
  - Use `fullPath` for include matching; fix leading slash mismatches for glob patterns.
  - Guard invalid regex in `matchGlob` (try/catch).
- `app/api/filesystem/list/route.ts`
  - Add bounds/cleanup for `requestTracker` map to prevent unbounded growth.
